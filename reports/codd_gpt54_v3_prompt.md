### CPDD技術アーキテクチャ（参考）

コアデータ構造: CEG（Conditioned Evidence Graph）
- 条件付き・証拠付き・時間付きのproperty graph
- ノード: ソースコード、設定、データ定義、ドキュメント、暗黙知
- エッジ: 5種の意味論（Structural, Resolution, Behavioral, Validation, Governance）
- 各エッジに有効条件（environment, tenant, feature_flag, time_window, data_predicate）

Change Calculus: 変更原子 × 関係種 × 条件 → アクション帯域（Green/Amber/Gray）
- Green: 自動更新可能（高precision）
- Amber: 要レビュー（影響の可能性あり）
- Gray: 到達可能だがノイズの可能性あり

既存ツールとの差分:
- grep/LSP/Aider: Structural層のみ
- CPDD目標: Structural + Resolution + Behavioral + Validation + Governance全5層
- Resolution〜Governanceが未開拓のフロンティア

## 補足: GPT-5.4 Pro Heavy Mode のプロンプトパターン解説

このV3プロンプトは以下の4パターンを組み合わせている:

1. **Output Contract**: `<output_contract>`で出力構造を厳密に定義。Heavy modeは長時間の思考後に構造化出力を生成するのが得意
2. **dig_deeper_nudge**: 各セクション末尾に自己検証を要求。Thinkingフェーズでの深掘りを促進
3. **Verification Loop**: `<verification_loop>`で全体の整合性チェックを最後に要求
4. **Prior Analysis Summary**: 前回の結論を明示し、差分分析に集中させる

V1/V2との差分:
- V1: 22問の質問形式（広く浅く）
- V2: 前提修正 + 収束確認（結論の更新）
- **V3: 5つの盲点に絞って深掘り + 自己検証ループ（狭く深く）**
